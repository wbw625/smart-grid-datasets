## ERENO

### Link
- [Kaggle dataset page](https://www.kaggle.com/datasets/sequincozes/ereno-iec61850-ids/data)
- [ERENO framework thesis / documentation source](https://www.ppgee.ufrgs.br/publicacoes/teses/Tese_SEQ.pdf)

### Introduction
ERENO is an IEC 61850 cyber-physical intrusion detection dataset suite generated by the ERENO framework. Public descriptions indicate that it contains **7 attack use cases plus 1 legitimate case**, and the released tabular datasets contain **69 features** combining **GOOSE communication features** and **SV-derived electrical/physical features**. In the dataset design, each row is aligned to an **SV message plus the most recent GOOSE message**, so it can be used directly as a supervised tabular classification dataset without first reconstructing sliding windows.

### Specific data fields
The official feature list exposed in the ERENO thesis includes:

- **SV basic / physical-value fields (1-13):**  
  `time`, `isbA`, `isbB`, `isbC`, `ismA`, `ismB`, `ismC`, `vsbA`, `vsbB`, `vsbC`, `vsmA`, `vsmB`, `vsmC`
- **SV enriched fields (14-37):**  
  `isbARmsValue`, `isbBRmsValue`, `isbCRmsValue`, `ismARmsValue`, `ismBRmsValue`, `ismCRmsValue`, `vsbARmsValue`, `vsbBRmsValue`, `vsbCRmsValue`, `vsmARmsValue`, `vsmBRmsValue`, `vsmCRmsValue`, `isbATrapAreaSum`, `isbBTrapAreaSum`, `isbCTrapAreaSum`, `ismATrapAreaSum`, `ismBTrapAreaSum`, `ismCTrapAreaSum`, `vsbATrapAreaSum`, `vsbBTrapAreaSum`, `vsbCTrapAreaSum`, `vsmATrapAreaSum`, `vsmBTrapAreaSum`, `vsmCTrapAreaSum`
- **GOOSE basic / protocol fields (38-59):**  
  `t`, `gooseTimestamp`, `sqNum`, `stNum`, `cbStatus`, `frameLen`, `ethDst`, `ethSrc`, `ethType`, `gooseTTL`, `gooseAppid`, `gooseLen`, `TPID`, `gocbRef`, `datSet`, `goID`, `test`, `confRev`, `ndsCom`, `numDatSetEntries`, `APDUSize`, `protocol`
- **GOOSE enriched / delta fields (60-69):**  
  `stDiff`, `sqDiff`, `gooseLengthDiff`, `cbStatusDiff`, `apduSizeDiff`, `frameLengthDiff`, `timestampDiff`, `tDiff`, `timeFromLastChange`, `delay`

### Suggested usage
This dataset is suitable for **mixed cyber + physical** IEC 61850 intrusion detection research. A cleaner experimental design is to benchmark at least three settings:

1. **GOOSE-only**
2. **Basic SV + GOOSE**
3. **All features**

Because the engineered features are strong, using all 69 features directly may make the task relatively easy. Since each row is already a supervised sample, both classical machine learning and LLM-style tabular fine-tuning can be reproduced without building a sliding-window pipeline first.

### Related papers and studies

- **ERENO: A Framework for Generating Realistic IEC-61850 Intrusion Detection Datasets**  
  Link: [IEEE page](https://www.computer.org/csdl/journal/tq/2024/04/10339874/1SBLdicrdTi)  
  Related thesis: [PDF](https://www.ppgee.ufrgs.br/publicacoes/teses/Tese_SEQ.pdf)  
  Reported results show that feature enrichment/selection can greatly improve performance. One highlighted example is **J48 F1 improving from 52.24% to 99.46%** in the framework study.

- **Machine Learning-Based Feature Selection for Intrusion Detection Systems in IEC 61850-Based Digital Substations**  
  Link: [ResearchGate page](https://www.researchgate.net/publication/385539491_Machine_Learning-Based_Feature_Selection_for_Intrusion_Detection_Systems_in_IEC_61850-Based_Digital_Substations)  
  This paper explicitly uses ERENO and studies which features are most useful for intrusion detection.

## Dataset to Train Intrusion Detection Systems based on Machine Learning Models for Electrical Substations (SANDI-2024)

### Link
- [Zenodo dataset page](https://zenodo.org/records/15487636)
- [Dataset paper in Data in Brief](https://www.sciencedirect.com/science/article/pii/S2352340924011156)
- [Companion preprocessing repository](https://github.com/esguti/cybersecurity-datasets)

### Introduction
SANDI-2024 is a substation IDS dataset designed for machine-learning benchmarking in electrical substations. The official paper states that it was collected from **real substations in Iltsi (Ukraine) and Granada (Spain)** plus **laboratory testbeds in Zaragoza (Spain)**. It includes both **raw PCAP files** and **processed CSV files** for **IEC 61850** and **IEC 60870-5-104 (IEC104)** traffic, and the paper also mentions traffic involving **NTP** and **PTP** in the substation environment.

### Specific data fields
The dataset is distributed in two layers:

- **Raw capture layer:** PCAP files for benign and attack scenarios. The Zenodo page lists IEC 61850 benign captures such as `capture61850-attackfree.pcap`, `capture61850-attackfree_PTP.pcap`, and `capture61850-attackfree_normalfault.pcap`; attack captures such as `floodattack_*`, `fuzzyattack_*`, `replay`, and `ptpattack`; and IEC104 benign/attack captures such as `capture104-attackfree.pcap`, `dosattack`, `floodattack`, `fuzzyattack`, `iec104starvationattack`, `mitmattack`, `ntpddosattack`, and `portscanattack`.
- **Processed CSV layer for IEC104:** generated with **CICFlowMeter**, so the columns follow the standard flow-level schema such as `Flow ID`, source/destination IP and port, `Protocol`, `Timestamp`, `Flow Duration`, forward/backward packet counts and byte totals, packet-length statistics, inter-arrival-time statistics, TCP flag counters, header lengths, subflow features, active/idle statistics, plus `Label`.
- **Processed CSV layer for IEC 61850:** the preprocessing script extracts many Wireshark/tshark fields including `frame.*`, `eth.*`, `goose.*`, `sv.*`, `ip.*`, `tcp.*`, and `ptp.*`, plus a final `Label`. Examples include:
  - `frame.time_delta`, `frame.len`, `frame.protocols`
  - `eth.dst`, `eth.src`, `eth.type`
  - `goose.appid`, `goose.length`, `goose.gocbRef`, `goose.timeAllowedtoLive`, `goose.goID`, `goose.t`, `goose.stNum`, `goose.sqNum`, `goose.confRev`, `goose.ndsCom`, `goose.numDatSetEntries`
  - `sv.appid`, `sv.length`, `sv.svID`, `sv.datSet`, `sv.smpCnt`, `sv.confRev`, `sv.refrTm`, `sv.smpSynch`, `sv.smpRate`
  - `ip.src`, `ip.dst`, `tcp.srcport`, `tcp.dstport`, `tcp.seq`, `tcp.ack`
  - many `ptp.*` fields

### Suggested usage
This dataset is best treated as a **multi-view benchmark** rather than a single task. Recommended experimental settings include:

1. **IEC104-only flow classification**
2. **IEC61850-only packet/field classification**
3. **Raw-PCAP custom feature extraction**
4. **Cross-protocol transfer** between IEC104 and IEC61850

Because the processed CSVs already exist, the dataset is convenient for supervised fine-tuning. Because raw PCAPs are also available, it is also suitable for custom protocol-aware feature engineering.

### Related papers and studies

- **A dataset to train intrusion detection systems based on machine learning models for electrical substations**  
  Link: [ScienceDirect](https://www.sciencedirect.com/science/article/pii/S2352340924011156)

## IEC 60870-5-104 Intrusion Detection Dataset

### Link
- [Zenodo dataset page](https://zenodo.org/records/7108614)

### Introduction
This is a dedicated IDS dataset for **IEC 60870-5-104 (IEC104)** traffic. The official Zenodo description states that it contains **labeled TCP/IP network-flow statistics** and **IEC104 flow statistics** for **12 IEC104 cyberattacks**, along with the underlying **PCAP files**. The attacks include both **unauthorized-command** scenarios and **DoS** scenarios, and Zenodo also provides a **balanced train/test CSV package**.

### Specific data fields
The dataset provides two processed tabular views:

- **TCP/IP flow-statistics CSVs:** generated with CICFlowMeter. Field families include `Flow ID`, source/destination IP and port, `Protocol`, `Timestamp`, `Flow Duration`, forward/backward packet counts and lengths, packet-length max/min/mean/std/variance, flow bytes/s and packets/s, inter-arrival-time statistics, PSH/URG flag counters, TCP flag counts, header lengths, subflow statistics, active/idle statistics, and `Label`.
- **IEC104 flow-statistics CSVs:** generated by a custom IEC104 parser. The published readme exposes a protocol-aware schema including:
  - flow identifiers: `Flow ID`, `Protocol`, source/destination IP and port
  - activity, idle, and IAT statistics
  - flow rates and APDU-length statistics
  - forward/backward packet counters
  - protocol-message counters for `IEC104_I_Message_SeqIOA`, `IEC104_I_Message_SingleIOA`, `IEC104_S_Message`, and `IEC104_U_Message`
  - TCP flag counters
  - subflow and bulk statistics
  - initial TCP window sizes and TCP header lengths
  - a long series of **cause-of-transmission (COT)** counters such as:
    - `cot=1_periodic,cyclic`
    - `cot=2_background interrogation`
    - `cot=3_spontaneous`
    - `cot=4_initialized`
    - `cot=5_interrogation`
    - `cot=6_activation`
    - `cot=7_confirmation activation`
    - `cot=8_deactivation`
    - `cot=9_confirmation deactivation`

### Suggested usage
This is a good **flow-level benchmark** for protocol-specific OT intrusion detection. A strong evaluation design is to compare:

1. **TCP/IP generic flow features only**
2. **IEC104 parser features only**
3. **Fusion of both feature views**

Because the dataset publisher already provides balanced train/test CSV files, those files are a sensible baseline starting point before trying custom splits.

### Related papers and studies

- **Breaching the Defense: Investigating FGSM and CTGAN Adversarial Attacks on IEC 60870-5-104 AI-enabled Intrusion Detection Systems**  
  Link: [ACM page](https://dl.acm.org/doi/10.1145/3600160.3605163)  
  Accessible version: [ResearchGate page](https://www.researchgate.net/publication/373483589_Breaching_the_Defense_Investigating_FGSM_and_CTGAN_Adversarial_Attacks_on_IEC_60870-5-104_AI-enabled_Intrusion_Detection_Systems)  
  The reported baseline table includes **Random Forest accuracy 0.8244 and F1 0.8098**, while the shown MLP baseline reaches **accuracy 0.6908 and F1 0.6852**.

- **A method for smart grid intrusion detection through explainable deep learning**  
  Link: [Springer page](https://link.springer.com/article/10.1007/s11416-025-00549-1)  
  This paper also uses the IEC104 dataset. The visible performance table reports results around **0.900** on the main precision/recall/F1 entries for the highlighted model.

## DNP3 Intrusion Detection Dataset

### Link
- [IEEE DataPort dataset page](https://ieee-dataport.org/documents/dnp3-intrusion-detection-dataset)
- [Dataset overview / readme host page](https://pradoglougrammatikis.com/portfolio-items/dnp3-intrusion-detection-dataset/)

### Introduction
This dataset targets IDS research for **DNP3-based SCADA** traffic. The official description and readme state that it includes **labeled TCP/IP network-flow statistics**, **DNP3 flow statistics** from a custom parser, and the corresponding **PCAP files**. The testbed contains **9 attack scenarios**:

- Disable Unsolicited Messages
- Cold Restart
- Warm Restart
- Enumerate
- Info
- Initialize Data
- MITM-DoS
- Replay
- Stop Application

The release also includes **balanced training/testing CSV files**.

### Specific data fields
Two main processed views are provided:

- **TCP/IP flow-statistics CSVs:** standard CICFlowMeter-style columns such as `Flow ID`, source/destination IP and port, `Protocol`, `Timestamp`, `Flow Duration`, total forward/backward packets, packet-length statistics, IAT statistics, TCP flag counters, header lengths, subflow features, active/idle statistics, and `Label`.
- **DNP3 flow-statistics CSVs:** the readme exposes protocol-aware fields including:
  - identifiers: `Flow ID`, source/destination IP and port, `Protocol`, `Date`
  - directional packet counts: `TotalFwdPkts`, `TotalBwdPkts`
  - total payload lengths by layer and direction: `TotLenfwdDL`, `TotLenfwdTR`, `TotLenfwdAPP`, `TotLenbwdDL`, `TotLenbwdTR`, `TotLenbwdAPP`
  - packet-length statistics for DNP3 data-link / transport / application layers
  - throughput and IAT statistics by layer
  - directional header-length fields
  - `fwdPkts/sec`, `bwdPkts/sec`
  - active/idle statistics
  - communication-direction / semantic fields such as:
    - `frameSrc`
    - `frameDst`
    - `TotPktsInFlow`
    - `firstPacketDIR`
    - `mostCommonREQ_FUNC_CODE`
    - `mostCommonRESP_FUNC_CODE`
    - `corruptConfigFragments`
    - `deviceTroubleFragments`
    - `deviceRestartFragments`
    - `pktsFromMASTER`
    - `pktsFromSLAVE`
  - `Label`

### Suggested usage
This is one of the stronger public datasets for **protocol-aware OT intrusion detection**. Recommended study settings include:

1. **Binary attack detection**
2. **9-way attack classification**
3. **Feature-view comparison**: generic TCP/IP flow features vs. DNP3 parser features vs. fused features

Because balanced train/test files and multiple parser-timeout variants are included, the dataset is also useful for robustness studies on preprocessing choices.

### Related papers and studies

- **A two-stage cyber attack detection and classification system for smart grids**  
  Link: [ScienceDirect](https://www.sciencedirect.com/science/article/pii/S2210537925000228)  
  The paper reports **F1 = 0.9976** for attack detection and **F1 = 0.9883** for attack-type classification on the DNP3 dataset.

- **An effective intrusion detection scheme for Distributed Network Protocol 3 (DNP3) applied in SCADA-enabled IoT applications**  
  Link: [ScienceDirect](https://www.sciencedirect.com/science/article/pii/S2405959523001483)  
  The paper reports **99.56% accuracy** on multi-class classification over nine DNP3 attack types.

## ICS Cyber Attack Datasets – Power System Datasets

### Link
- [Official UAH / Tommy Morris page](https://sites.google.com/a/uah.edu/tommy-morris-uah/ics-data-sets)

### Introduction
This is the well-known **Power System Datasets** collection from the ICS Cyber Attack Datasets page. Public descriptions and later papers describe **37 power-system event scenarios** grouped into:

- **28 attack events**
- **8 natural events**
- **1 no-event case**

This is a **fusion dataset**, not a pure network-flow dataset. It combines **time-synchronized PMU/relay electrical measurements** with **cyber/logging data**. Later works also note that the released tabular files were **randomly sampled at 1%**, which matters when deciding whether adjacent rows should be treated as sequential time steps.

### Specific data fields
From the public descriptions, the following structure is confirmed:

- **128 features** in total
- **116 columns** are **PMU/relay measurement columns**
- **12 columns** are **control-panel logs, Snort alerts, relay logs, and the marker/target column**

The 116 measurement features come from **four PMU/relays**, each contributing **29 measurement features**. Publicly visible measurement groups include:

- `PA1:VH-PA3:VH` — phase A-C voltage phase angle
- `PM1:V-PM3:V` — phase A-C voltage magnitude
- `PA4:IH-PA6:IH` — phase A-C current phase angle
- `PM4:I-PM6:I` — phase A-C current magnitude
- `PA7:VH-PA9:VH` — positive/negative/zero-sequence voltage phase angle
- `PM7:V-PM9:V` — positive/negative/zero-sequence voltage magnitude

Example concrete column names visible in later analyses include `R1-PA1:VH`.

### Suggested usage
This dataset is best used as a **tabular cyber-physical classification dataset**, not as a pure network IDS benchmark. Because public descriptions state that the released table was already **randomly sampled at 1%**, each row should generally be treated as an **independent sample** unless the original temporal order is reconstructed from raw data. It is especially suitable for comparing:

1. **Physical-only models**
2. **Cyber/log-only models**
3. **Fused-feature models**

### Related papers and studies

- **Developing a Hybrid Intrusion Detection System Using Data Mining for Power Systems**  
  Link: [DOI page](https://doi.org/10.1109/TSG.2015.2409775)

- **Classification of Disturbances and Cyber-attacks in Power Systems Using Heterogeneous Time-synchronized Data**  
  Link: [DOI page](https://doi.org/10.1109/TII.2015.2420951)

- **Power system events classification using genetic algorithm based feature weighting technique for support vector machine**  
  Link: [ScienceDirect](https://www.sciencedirect.com/science/article/pii/S2405844021000414)  
  This paper explicitly uses the power-system dataset and reports strong classification performance, with the proposed GA-SVM method outperforming comparison models.

- **Intrusion Detection in SCADA-based Power Grids**  
  Link: [PDF](https://dcsi.cs.dal.ca/wp-content/uploads/2021/07/21.pdf)  
  This paper reports best accuracies of **97.66%, 96.50%, 94.12%, and 92.46%** across four categories of the power-system datasets while using a reduced feature subset.

## Network Traffic Dataset IEC-61850 Cyber Attacks

### Link
- [Kaggle dataset page](https://www.kaggle.com/datasets/johnatandeoliveira/network-traffic-dataset-iec-61850-cyber-attacks)

### Introduction
This is a **network-traffic dataset for IEC 61850 cyber attacks**. The public Kaggle description presents it as **realistic GOOSE traffic** collected under **four different cyberattack types**. Based on the available public description, it is a **GOOSE packet/frame-level** dataset rather than a flow-level dataset or a pure physical-process dataset.

### Specific data fields
The publicly accessible preview pages do **not expose a reliable full column/header list**, so the exact CSV schema cannot be safely confirmed from the source page alone. What is publicly visible confirms that this is a **tabular CSV dataset of GOOSE network traffic under four attack types**. The exact field names should be checked from the downloaded CSV files before designing a feature map.

### Suggested usage
This dataset can be used as a **quick packet-level supervised benchmark** for GOOSE attack classification, especially when a simpler alternative to multi-view cyber-physical datasets such as ERENO is needed. Before training, the exact label column and any potential leakage features should be verified. It may also be useful for **transfer-learning experiments across public GOOSE datasets**.

## EPIC Attack Datasets (EPIC-A / EPIC-B)

### Link
- [GitHub repository](https://github.com/smartgridadsc/EPIC_Attack_Datasets)
- [Dataset paper page](https://ieeexplore.ieee.org/document/10738043)

### Introduction
EPIC is a **high-fidelity smart-grid cyber-physical dataset family** built around the EPIC testbed. The public paper describes it as a hardware-based smart-grid security dataset covering **false-data injection attacks (FDIA)** and **time-delay attacks (TDA)** in operations such as **generator synchronization** and **reverse power prevention**. The public GitHub repository contains **EPICA_Dataset** and **EPICB_Dataset** folders with labeled attack scenarios. Public descriptions also state that the dataset contains **both network data and process data** from IEDs monitoring electrical parameters.

### Specific data fields
The following information is publicly confirmed:

- many per-run **labeled complete CSV files** are stored under **EPICA_Dataset** and **EPICB_Dataset**
- the attack scripts/descriptions involve smart-grid control and physical variables such as:
  - manipulated **boolean sync commands**
  - **phase-angle values**
  - **reverse-power indicators**
  - **VSD speed settings**
- public summaries describe the data as containing **both network features and process/physical features**

However, the accessible public previews do **not reliably expose the exact CSV header row**, so a full field-by-field schema cannot be safely listed without downloading the files and inspecting them locally.

### Suggested usage
EPIC is better suited to **cyber-physical anomaly detection** and **sequence/window-based modeling** than to a plain row-independent flow benchmark, because the attacks evolve through physical control behavior. Suitable uses include:

1. **FDIA/TDA detection**
2. **Windowed sequence classification**
3. **Cross-scenario robustness evaluation**

The labeled CSV files in the repository also make supervised learning possible.
